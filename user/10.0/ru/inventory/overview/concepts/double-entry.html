Оригинал на GitHub

При управлении запасами с помощью правила двойной записи не существует поступления, ухода (несоответствие запасов) или трансформации. Вместо этого все операции являются движениями запасов между расположениями (возможно виртуальными).

Операции

Движение складских запасов представляет собой транзит товаров и материалов между расположениями.

Производственный заказ
Израсходовано:
2 Колеса: Склад → Производство
1 Мотоциклетная рама: Склад → Производство
Произведено:

1 Мотоцикл: Производство → Склад

Конфигурация:
Склад: расположение откуда может быть инициирован Производственный Заказ
Производство: в форме товара, поле "Расположение Производства"
Прямая поставка

1 Мотоцикл: Поставщик → Клиент

Конфигурация:
Поставщик: в форме товара
Потребитель: указан в заявке
Доставка клиенту
Подбор

1 Мотоцикл: Склад → Зона упаковки

Упаковать

1 Мотоцикл: Зона упаковки → Внешняя площадка

Доставка

1 Мотоцикл: Внешняя площадка → Потребитель

Конфигурация:
на маршруте подбор+упаковка+доставка для склада
Внутри-Складское перемещение
Перемещение:
1 Мотоцикл: Склад 1 → Впути
1 Мотоцикл: Впути → Склад 2
Конфигурация:
Склад 2: расположение, откуда было инициировано перемещение
Warehouse 1: на маршруте перемещения
Испорченный товар (Лом, Бой)

1 Мотоцикл: Склад → Лом

Конфигурация:

Лом: расположение лома при возникновении необходимости утилизации

Управление запасами (Inventory)
Недостача

1 Мотоцикл: Склад → Недостача

Излишки

1 Мотоцикл: Недостача → Склад

Конфигурация:

Недостача: "Местоположение запасов" поле в форме товара

Оприходование товара
1 Мотоцикл: Поставщик → Прием товара
1 Мотоцикл: Прием товара → Запасы
Конфигурация:
Поставщик: в наряде на поставку графа "поставщик"
Прием товара: поле "назначение" в наряде на поставку

Анализ

Анализ складских запасов может использовать количественный учет или суммовой (= количество товаров * на их стоимость)

Для каждого расположения запасов можно проанализировать несколько точек данных:

  • inventory valuation
  • value creation (difference between the value of manufactured products and the cost of raw materials used during manufacturing) (negative)
  • value of lost/stolen products
  • value of scrapped products
  • value of products delivered to clients over a period
  • value of products received from suppliers over a period (negative)
  • value of products in transit between locations

Закупки и Правила закупок

Закупка представляет собой запрос на определенное количество товаров в определенном расположении. Зкупки могут создаваться вручную или формироваться автоматически с помощью:

Новых заявок
Эффект

Закупка создается в расположении клиента для каждого продукта, заказанного этим клиентом (вы должны предоставить данные о потребителе)

Конфигурация

Расположение закупки: в форме клиента "Расположение Клиента"(опционально)

Правило минимального количества запасов
Эффект

Закупка создается в расположении правила.

Конфигурация

Расположение закупки: в форме правила поле "Расположение"

Правила закупок
Эффект

Новая закупка создается в расположении источника правила

Procurement rules describe how procurements on specific locations should be fulfilled e.g.:

  • где продукт должен поступать из (источник расположения)

  • является ли закупка ПНЗ или ПНС

Маршруты

Правила закупок сгруппированы по маршрутам. Маршруты определяют пути, которым должен следовать продукт. Маршруты могут быть применены или нет, в зависимости от товаров, позиций в заявке от клиента, склада, ...

Для выполнения закупок система будет искать правила, относящиеся к описанным маршрутам (в порядке выстроенных приоритетов):

Склады

Пример маршрута для склада: Подбор → Упаковка → Доставка

Список для подбора:

Зона подбора → Зона упаковки

Список упаковки:

Зона упаковки → Ворота А

Наряд на доставку:

Ворота А → Клиент

Маршруты, описывающие порядок организации вашего склада, должны быть определены на складе

Продукт

Пример маршрута товара: Контроль Качества

Прием товара:

Поставщик → Место доставки

Подтверждение:

Место доставки → Контроль качества

Хранилище:

Контроль Качества → Хранилище

Категория товара

Пример маршрута категории товара: кросс-докинг

Прием товара:

Поставщик → Место доставки

Кросс-Док:

Место доставки → Место отгрузки

Доставка:

Место отгрузки → Клиент

Позиция в заявке

Пример маршрута для позиции в заявке: Дроп-шиппинг

Заказ:

Поставщик → Клиент

Правила выталкивания

Правила выталкивания запускаются, когда товар поступает в конкретное расположение. Они автоматически перемещают продукт в новое расположение. Когда нужно использовать правила выталкивания, зависит от соответствующих маршрутов.

Контроль качества

  • Товар попадает на Место доставки

  • Движение 1: Место доставки → Контроль Качества

  • Движение 2: Контроль качества → Хранилище

Складское перемещение

  • Товар переходит в состояние Впути

  • Движение: Впути → Склад 2

Группы закупок

Маршруты и правила определяют движения запасов. Для каждого правила предоставляется тип документа:

  • Подбор

  • Упаковка

  • Наряд на доставку

  • Заказ на закупку

  • ...

Перемещения группируются в пределах одного и того же типа документа, если их группа закупок и расположения одинаковы.

По заявке от клиента создается группа закупок таким образом, чтобы заказы на доставку и подбор из одной и той же заявки были сгруппированы. Но вы также можете указать определенным группам работать по другим правилам. (например, группировать закупки отдельных продуктов вместе)

  • Dapibus ac facilisis in
  • Morbi leo risus
  • Porta ac consectetur ac
  • Vestibulum at eros
